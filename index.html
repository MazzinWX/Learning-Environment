De l'utilisation de Git, GitHub, MarkDown et autres outils

Fonctions essentielles de Git:

  • git add "nom du fichier" // Pour mettre à l'index
  • git commit -m "+message" // Validation de "add"
  • git push "Envoi au dépôt distant" // De local à distant
  • git pull "Mise à jour du dépôt local" // De distant à local

Autres fonctions utiles:

  • git branch "nom de la branche" // Crée une branche à l'endroit du HEAD
  • git checkout "nom de la branche voulue" // Place le HEAD à l'endroit voulu

Utilité de GitHub:

Permet de se briefer sur les différents éléments à gérer d'un projet.

Plateforme participative utilisée comme un gestionnaire de projet.

Utilisation de MarkDown:

Plusieurs formatages utiles et faciles à appliquer, utilisant des raccourcis de balises HTML

  • " * " ou " _ " // Italique
  • " ** " ou " __ " // En Gras
  • " 4 espaces " ou " 1 Tab" // place en mode "Code source" / Indentation
  • " # " ou " ## " // Comparable à H1 et H2
  • " + " sur deux lignes ou plus // Liste à puces
  • " > " // Citation
  • " ` " sur une ligne // Insérer du code sur une ligne
  • " ["Description"]("lien du site") // Créer une phrase hypertexte
  • " ![]() " // Même chose que les liens, mais pour les images
